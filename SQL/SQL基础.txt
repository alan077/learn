SQL Server的组件
    数据库引擎
        SQL Server数据库引擎有时称为关系引擎，它是SQL Server的核心。它是处理关系数据库的所有工作的组件。SQL是一种描述性语言，即它仅向引擎描述问题，引擎从哪里开始运行
        关系引擎捏有许多关键进程和组件：
            Algebrizer(代数器)：检查语法并把查询转化为以下组件使用的内部表达式
            Query Optimizer(查询优化器)：SQL Server的查询优化器根据不同类型查询--执行操作的成本决定如何最好地处理查询。利用Management Studio或SQL Profiler以图形方式或在XML中查看估计的和实际的查询-执行计划
            Query Engine(查询引擎)或Query Processor(查询处理器)：根据查询优化器生成的计划进行查询
            Buffer Manager(缓冲区管理器)：分析正在使用的数据页，从数据文件中预提取数据并存入内存，从而减少对磁盘I/O性能的依赖性
            Checkpoint(检查点)：将内存中脏数据页(修改的数据页)写入数据文件的进程
            Resource Monitor(资源监视器)：通过响应内存压力，智能地从缓存区删除较早的查询计划来优化查询计划
            Lock Manager(锁管理器)：用锁的大小动态管理锁定的范围以平衡所需的数量
            SQLOS：SQL Server会占资源，为此需要直接控制可用资源(内存、线程、I/O请求等)。仅把资源管理工作留给Windows对于SQL Server还不够复杂。SQL Server有自己的操作系统层(SQLOS)，它负责管理所有内部资源
    服务
        下列组件是用于控制SQL Server或与之通信的客户端进程
            SQL Server Agent
                Server Agent是一个可选进程，运行时执行SQL作业并处理其他自动任务。系统启动时可以配置该进程为自动运行，或可以从SQL Server配置管理器或Management Studio的Object Explorer(对象资源管理器)中启动
            数据库邮件
                数据库邮件(Database Mail)组件使SQL Server能通过SMTP将邮件发送到外部邮箱。通过SQL Server中的多种渠道生成邮件，包括T-SQL代码、作业、报警、集成服务器和维护计划
            Microsoft分布式事务处理协调器(MSDTC)
                分布式事务处理协调器(DTC)是一项跨越多个SQL Server处理双向事务提交的过程，可从Windows的计算机管理/服务中启动DTC。如果应用程序经常使用分布式事务，则应该在启动操作系统时启动DTC
                
数据架构
    信息架构原则：信息是组织的资产，必须根据其价值和服务进行组织、清点、保护，并以可用的格式准备随时用于日常操作，以便于当今及今后的个体、组织进行分析和处理
    数据库目标：适用性、可扩展性、数据库完整性和性能，可用性和安全性是功能的实现而非设计

逻辑查询处理    
    在SQL Server中负责生产实际工作计划(执行计划)的组件是查询优化器。以何种顺序访问表、使用寿命访问方法和索引、应用那种联接算法等都是由查询优化器来决定的。优化器会生产多个有效的执行计划，并从中选择一个开销最低的计划。逻辑查询处理中的各个阶段都有其非常特定的顺序。相反，优化器却经常可以在它生成的物理执行计划中走捷径，当然，只有在保证结果集正确的前提下，优化器才会走捷径；换句话说，走捷径所得到的结果集必须与遵循逻辑处理的各个阶段所得到的结果集相同，例如，为了使用索引，优化器可能会决定使用一个比逻辑处理所规定的筛选器快得多的筛选器
